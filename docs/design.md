# 设计规范

## 视觉方向
- Apple 气质启发，不做 1:1 复刻
- 主界面偏 `Notes / Quick Look / AirDrop` 的工具感
- 用备忘录式列表、纸面感画布和克制动效建立安静的内容优先体验

## 主题 Token
- 浅色：冷白、雾灰、石墨灰
- 深色：蓝灰黑、石板灰
- 强调色：低饱和蓝，仅用于焦点、选中、链接和主要按钮
- 圆角：14 / 20 / 28 三档
- 阴影：柔和投影，不使用厚重浮夸的黑影
- 动效：160ms / 220ms，优先 `opacity + transform`

## 组件约束
- 按钮：主按钮使用低饱和蓝，次按钮使用轻描边与半透明底
- 顶部工具按钮：统一为圆形图标按钮，管理态激活时允许用状态点和轻强调底色提示
- 输入框：弱边框、柔和背景、聚焦蓝色描边
- 分段控件：高密度 Apple 风格 segmented control
- 共享流：收敛为单一备忘录列表，标题、摘要、时间和类型必须清晰分层
- 详情面板：参考 Quick Look 的大预览，并为便签提供可直接编辑的正文画布
- 空状态：少字数、高留白、轻图形，不堆教程
- 弹窗：统一使用轻玻璃感居中浮层，移动端改为底部弹层，避免每个入口各自实现一套交互

## 详情层级
- 内容区优先：图片、文字、PDF 预览或正文编辑必须占据详情面板的主视觉区域
- 编辑区次级：标题输入、保存、取消和删除位于内容画布前后，避免和预览混杂
- 属性区最轻：格式、大小、来源、哈希等信息采用更轻的定义列表排布
- 桌面端详情列允许比普通侧栏更宽，避免内容画布被属性区域挤压

## 管理态约束
- 匿名访问者仅可上传、查看和下载
- 管理态解锁后可编辑所有共享项标题，可编辑便签正文，并保留删除能力
- 保存方式采用显式提交，不做自动保存

## 响应式原则
- 桌面端采用顶部操作区 + 双列结构：共享流 / 详情
- 移动端改为单列堆叠，详情面板跟随共享流向下展开
- 顶部工具栏允许自动换行，快捷新建弹窗在窄屏下改为单列内容布局
- 不隐藏核心上传、预览、编辑、下载、删除能力，只调整布局密度
